## 1. 产品目标（3 个清晰、正交的目标）

### 目标 1：降低记账摩擦，突破留存天花板
**问题对标**：90% 用户三个月内放弃记账，80% 因"太麻烦"放弃，每增加一步操作放弃概率增加 40%。
**目标定义**：通过 AI 账单批量导入、语音/对话式极速记账，将单笔记账时间从行业平均 30-60 秒降至 **5 秒以内**，月度账单整理时间降至 **30 秒以内**。目标实现 D7 留存≥55%、D30 留存≥35%、连续记账 30 天用户比例≥30%（行业基准<8%）。

### 目标 2：从"记录"升级到"建议"，建立 AI 财务洞察壁垒
**问题对标**：用户痛点"有饼图无解读""记账≠存钱""只记不会用"。竞品中 AI 消费洞察分析仅 1 款实现，AI 财务对话助手完全空白。
**目标定义**：每月自动生成自然语言财务洞察报告，提供可执行建议。目标报告阅读率≥60%、建议采纳率≥25%、用户每月通过建议平均优化支出≥500 元。

### 目标 3：占领家庭财务协同蓝海
**问题对标**：AI 家庭财务规划助手完全空白，家庭财务目标协同管理高度空白。
**目标定义**：提供家庭共同账本+个人子账本的多层账本体系，支持 3-6 人家庭共享。目标家庭账本渗透率≥40%，家庭用户 D90 留存≥50%。

## 2. 用户故事（5 个场景）

**US-1（新婚双薪夫妻）**：作为新婚夫妻，我希望能够共同管理家庭账本的同时保留各自的个人子账本，以便在保持财务透明的基础上尊重彼此的个人消费隐私，并共同追踪我们的购房储蓄目标。

**US-2（有孩家庭 CFO）**：作为负责家庭财务的妈妈，我希望每月能一键导入支付宝和微信账单并由 AI 自动分类，以便用不到 30 秒完成全家月度账目整理，而非逐笔手动记录上百条消费。

**US-3（三代同堂家庭支柱）**：作为同时承担房贷、子女教育和父母养老的家庭支柱，我希望看到一个全局家庭财务仪表盘和 AI 财务洞察报告，以便清晰了解三重压力下的现金流健康度，并获得"下一步该做什么"的可执行建议。

**US-4（独居职场人）**：作为经常情绪性消费的独居职场人，我希望预算提醒是温和的鼓励而非红色警告，并与我的旅行储蓄心愿目标关联，以便在被提醒时感到的是"离目标更近一步"的动力，而非负罪感。

**US-5（效率优先的记账新手）**：作为刚开始记账的新手，我希望通过语音说出"午饭花了 28 块"就能完成记账，AI 自动分类到"餐饮"并正确识别金额，以便在 5 秒内完成记账而无需打开多个页面手动选择分类。

## 3. 用户研究洞察（来自瑞思）

### 3.1 核心痛点（按严重程度排序）

**P0 级痛点（致命级——直接导致用户流失）：**

1. **记账坚持率极低：90% 用户三个月内放弃**
   - 四大放弃类型：期望偏差型（记账≠存钱）、负反馈型（红色警告→负罪感→逃避）、工具失灵型（AI 分类不准、同步不全）、价值迷失型（有饼图无解读）
   - 根本原因：放弃不是"懒"，而是对"失败产品体验"的合理"用脚投票"

2. **手动记账是最大摩擦，而非分类**
   - 用户账单"早就记好了"——微信支付和支付宝账单已记录 90%+ 消费
   - 用户需要的不是"再记一遍"，而是"把已有数据整理导入"

3. **多平台数据割裂：数据孤岛问题**
   - 消费分散在银行、支付宝、微信支付、信用卡，无产品实现"一键同步"

**P1 级痛点（严重——显著降低体验质量）：**

4. **记账与财务管理脱节：只记不会用** — 缺乏"消费结构是否健康"的判断基准和"下一步该做什么"的建议
5. **负反馈循环：预算功能变成"道德审判"** — 红色警告让用户逃离而非面对
6. **AI 分类准确率不足** — 纠正成本高于记账成本，信任崩塌

**P2 级痛点（改善级）：**

7. 分类体系僵化，不适合真实家庭场景
8. 多人记账协同难题（夫妻分账、代际差异、权限管理缺失）
9. 隐私安全顾虑

### 3.2 用户画像（4 个典型角色）

| 画像 | 特征 | 核心需求 | 产品对应 |
|------|------|----------|----------|
| 小林&小雅（25-32 岁） | 新婚双薪夫妻，家庭月收入 2-4 万 | 共同 vs 个人账户平衡、大额支出决策、目标储蓄追踪 | 家庭账本+个人子账本、心愿目标 |
| 陈姐（30-38 岁） | 有孩家庭 CFO，家庭月收入 1.5-3 万 | 教育支出规划、零散支出多、月光焦虑 | 账单批量导入、AI 洞察报告、温和预算 |
| 王哥（35-45 岁） | 三代同堂家庭支柱，家庭月收入 2-5 万 | 房贷+教育+养老三重压力、全局视图缺失 | 家庭财务仪表盘、支出预测 |

### 3.3 AI 技术应用三梯队

- **第一梯队（核心差异化，MVP 必做）**：账单智能导入、语音/对话式记账、AI 财务洞察报告、智能预算与温和预警
- **第二梯队（体验增强，v1.1-v1.2）**：OCR 票据识别、消费行为画像、家庭财务健康度评分、个性化省钱/理财建议
- **第三梯队（进阶，v2.0+）**：教育/医疗支出预测、风险保障缺口分析、家庭分账智能建议

### 4.3 差异化机会（⭐⭐⭐⭐⭐）

1. **AI 家庭财务规划助手** — 完全空白，最大蓝海
2. **AI 对话式财务顾问** — 完全空白
3. **家庭财务目标协同管理** — 高度空白
4. **AI 全平台账单聚合** — 高难度但高价值
5. **AI 消费预测+可执行规划** — 蓝海

### 4.4 竞争防御要点

- **防咔皮**：抢先建立家庭 AI 规划壁垒（咔皮可能从个人扩展到家庭）
- **防随手记**：2.2 亿用户基础，需在体验上拉开代差
- **防 YNAB**：若推中文版+AI，需在方法论本地化占先
- **防支付宝/微信**：强调跨平台聚合能力

## 7. 关键流程图 / UI 设计稿

### 7.1 账单智能导入流程

```
用户进入"导入账单"页面
→ 选择平台（支付宝/微信/银行/信用卡）
→ 上传文件（CSV/HTML/PDF，拖拽或选择）
→ 系统解析文件格式，提取交易记录
→ AI 逐笔自动分类（基于商户名/金额/时间特征）
→ 展示解析结果预览（已分类交易列表）
  → 用户可批量确认或逐笔纠正
  → 纠正操作反馈给 AI 学习
→ 确认导入 → 数据合并至家庭账本/个人子账本
→ 展示导入摘要（共 X 条，自动分类准确率 Y%）
```
**关键指标**：全流程<30 秒/月，自动分类准确率≥95%，纠正率<10%

### 7.2 语音/对话式记账流程

```
用户点击首页悬浮语音按钮（或输入框打字）
→ 语音模式："午饭 28 块" / 文字模式："打车 35 元去公司"
→ 语音识别转文字（<1 秒）
→ NLP 解析：提取金额/分类/时间/备注
→ AI 置信度判断：
  → ≥90%：直接生成记录，Toast 展示"已记：餐饮 -28 元 [撤销]"
  → <90%：弹出快速确认卡片，一键确认或修改
→ 记录写入账本 → 回到首页
```
**关键指标**：全流程<5 秒（语音）/ <3 秒（文字），识别准确率≥90%

### 7.3 AI 财务洞察月报流程

```
每月 1 日系统自动触发月报生成
→ AI 聚合上月全部交易数据
→ 多维度分析：
  1. 收支总览（收入/支出/结余/环比变化）
  2. 消费结构分析（各分类占比 vs 国标基准线，标记偏离项）
  3. 异常支出识别（单笔超均值 2 倍/某分类环比增 50%+）
  4. 可执行建议生成（基于异常项+用户画像+历史趋势）
  5. 心愿目标进度更新
→ 生成自然语言叙述报告 + 关键数据图表
→ 推送通知 → 用户点击进入
  → 报告分卡片展示，每条建议带"立即处理"按钮
  → 底部收集反馈："这条建议对你有帮助吗？"
```
**关键指标**：报告阅读率≥60%，建议采纳率≥25%

### 7.4 家庭账本创建与协同流程

```
用户选择"创建家庭账本" → 输入家庭名称
→ 系统生成邀请码/二维码/微信分享链接
→ 被邀请成员打开链接 → 注册/登录 → 加入家庭
→ 创建者设置成员角色（管理员/普通成员/只读）
→ 各成员记账时选择记入"共同账本"或"个人子账本"
→ 共同账本交易全员可见，个人子账本仅本人可见
→ 大额支出自动通知全体（可配置阈值）
→ 家庭财务仪表盘聚合展示全家数据
```
**关键指标**：家庭创建成功率≥80%，成员邀请转化率≥60%

### 7.5 智能预算与温和预警流程

```
用户首次进入预算功能
→ AI 分析历史 3 个月数据 → 推荐各分类预算金额
→ 用户确认或调整 → 关联心愿目标（如"日本旅行基金 ¥20000"）
→ 日常记账时实时更新预算执行进度
→ 预警触发（温和正向）：
  → 80%：推送"已用 80%，离目标还差一点，加油"
  → 100%：推送"本月预算已用完，但你的旅行基金已存了 ¥XXX"
  → 未超预算月末：推送庆祝"本月省了 ¥XXX，旅行基金完成度 XX%"
→ 月末预算执行报告（含下月 AI 调整建议）
```
**关键指标**：预警打开率≥70%，预算坚持率≥50%

### 7.6 新用户 Onboarding 流程

```
首次打开 App → 欢迎页（1 屏，<3 秒）
→ 注册/登录（手机号+验证码 或 微信登录，≤2 步）
→ 选择模式："个人记账" 或 "家庭记账"
→ 首次使用引导（非强制，可跳过）：
  1. "试试说一句话记账" → 引导语音记账
  2. "导入你的支付宝账单" → 引导账单导入
  3. "设置你的第一个心愿目标" → 引导预算+目标
→ 进入首页
```
**关键指标**：引导完成率≥70%，注册转化率≥85%

## 8. Non-goals（明确不做什么）

| 编号 | 不做内容 | 原因 |
|------|----------|------|
| NG-1 | 不做实时银行 API 直连同步 | 银行开放 API 接入门槛高、周期长，MVP 用文件导入替代 |
| NG-2 | 不做投资交易执行（买卖基金/股票） | 金融交易涉及牌照合规，非核心定位 |
| NG-3 | 不做社区/社交功能 | 运营成本高、与核心价值无关 |
| NG-4 | 不做广告变现 | 广告严重伤害体验，采用订阅制 |
| NG-5 | 不做全功能 PC 端 | 移动优先，PC 端仅做轻量 Web 版 |
| NG-6 | 不做手动复杂记账（如复式记账） | 目标用户是普通家庭，违背"降低摩擦"原则 |
| NG-7 | 不做独立支付/转账功能 | 非支付工具，不触碰支付牌照 |
| NG-8 | 不做保险/理财产品的销售推荐 | 避免利益冲突，AI 建议保持中立 |

### 停车场（好想法但不在当前范围）

- PK-1：儿童财商教育模块 → v2.5 评估
- PK-2：家庭财务日历（账单到期/缴费提醒） → v1.2 评估
- PK-3：Apple Watch / 智能手表快捷记账 → v1.1 评估
- PK-4：家庭财务年度盘点报告 → v2.0 规划
- PK-5：与银行/保险机构 API 对接的自动保障缺口扫描 → v2.5 评估

## 10. 待确认问题

### 产品方向类
1. **产品名称**：应用正式名称待定，建议突出"AI+家庭"双关键词
2. **目标平台优先级**：iOS/Android 同步上线，还是先做一个平台验证？
3. **变现模式确认**：免费基础版+付费 Pro 版分层？免费版功能边界在哪？

### 技术可行性类
4. **银行账单解析范围**：P0-03 计划覆盖"主流 3 家银行"（招行/工行/建行），格式差异是否可统一适配？
5. **语音识别方案**：自研 vs 接入第三方（腾讯云/科大讯飞）？路径建议使用成熟云 API
6. **AI 分类模型方案**：本地推理 vs 云端 API？数据隐私如何保证？
7. **家庭账本同步技术**：实时同步（WebSocket） vs 准实时同步（轮询）？

### 合规与安全类
8. **金融数据合规**：导入银行账单是否涉及金融数据合规审查？需要哪些资质？
9. **数据存储地域**：用户财务数据是否必须存储在中国境内服务器？
10. **未成年人保护**：家庭账本中若有未成年成员，数据保护有哪些额外要求？

### 用户体验类
11. **分类体系基准**：MVP 默认分类体系采用几级分类？（建议二级：一级 8 类对齐国标，二级可自定义）
12. **预算默认周期**：默认月度预算，是否同时支持周/双周/自定义周期？
13. **离线可用性**：无网络时是否支持记账？离线数据如何同步？

## ⚠️ 待确认 / 假设 / Non-goals

**待确认（需产品负责人决策）：**
- 产品正式名称与品牌定位
- iOS/Android 同步上线还是分平台验证
- 免费版与付费版的功能边界
- 语音识别与 AI 分类采用云服务还是自研
- 银行账单解析的合规审查需求

**关键假设：**
- 假设 7 人团队（前端 2+后端 2+AI 1+设计 1+测试 1）全职投入
- 假设 AI 分类模型通过规则+ML 混合方案可达 75%+ baseline
- 假设支付宝/微信账单导出格式在开发期间不发生重大变更
- 假设云 ASR/OCR 服务可用且延迟满足<5 秒要求
- 假设应用商店审核周期≤2 周

**Non-goals（明确不做）：**
- 不做实时银行 API 直连同步（MVP 用文件导入替代）
- 不做投资交易执行（买卖基金/股票，涉及牌照合规）
- 不做社区/社交功能（运营成本高、与核心价值无关）
- 不做广告变现（严重伤害体验，采用订阅制）
- 不做全功能 PC 端（移动优先，PC 端仅做轻量 Web 版）
- 不做复式记账（目标用户是普通家庭，违背"降低摩擦"原则）
- 不做独立支付/转账功能（非支付工具，不触碰支付牌照）
- 不做保险/理财产品销售推荐（避免利益冲突，AI 建议保持中立）
